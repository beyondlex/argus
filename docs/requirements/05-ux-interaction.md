# 多端用户体验与交互设计

## 1. CLI 端（快速验证与自动化测试）

面向脚本、自动化管道和核心逻辑验证。

### 1.1 命令集

```bash
# 单次扫描并保存快照
argus scan --path ~/Downloads --output ./snap.json

# 对比两个快照，按阈值过滤（支持人类可读格式如 50MB / 2.5GB）
argus diff --old ./snap-a.json --new ./snap-b.json --threshold 50MB

# 输出结构化报告（支持 text / json / markdown 格式）
argus diff --old ./snap-a.json --new ./snap-b.json --threshold 100MB --format json

# 模拟 AI 诊断：打印特征提取 Prompt（不调用任何 AI API）
argus explain --old ./snap-a.json --new ./snap-b.json --target-path ~/Library/Caches
```

### 1.2 阈值参数

`--threshold` 参数接受人类可读的体积格式，由 `argus-core` 提供解析函数 `parse_human_size`：

| 输入 | 实际值（字节） |
|------|---------------|
| `500B` | 500 |
| `10KB` | 10,240 |
| `50MB` | 52,428,800 |
| `2.5GB` | 2,684,354,560 |

默认值 `0` 表示显示所有变动（不过滤）。

> **Phase 1 实现**：`argus-core` 中实现 `parse_human_size` 函数，`argus-cli` 在解析参数时调用。

### 1.3 输出格式

- 文本格式（默认）：终端彩色打印，人类友好。
- JSON 格式：供 CI/CD 和自动化工具解析。
- Markdown 格式：可直接嵌入报告或文档。

### 1.4 退出码契约

CLI 退出码标准化，支持脚本化管道和 CI 集成：

| 退出码 | 含义 | 适用场景 |
|--------|------|---------|
| `0` | 成功，未发现超阈值变动 | `argus scan` / `argus diff` 正常完成 |
| `1` | 发现超阈值增长 | `argus diff` 存在 `size_delta >= threshold` 的条目；减少量不触发告警退出码 |
| `2` | 参数错误 | 非法路径、缺失必选参数、格式错误 |
| `3` | IO 错误 | 快照文件不存在、权限不足、写入失败 |
| `4` | 内部错误 | 快照损坏、数据不一致、反序列化失败 |

```bash
# CI 用法示例：检查 Downloads 是否新增 >100MB
argus diff --old snap_a.json --new snap_b.json --threshold 100MB
if [ $? -eq 1 ]; then
    echo "⚠️ 发现大变动，触发告警"
fi
```

## 2. TUI 端（Vim-like 核心极客版）

基于 `ratatui` 开发，布局分为四块。

### 2.1 布局与视觉

```
+--------------------------------------------------+
|  Argus v0.1.0              [?] Help  [Q] Quit    |
+--------------------------------------------------+
| 时间: [2026-06-01 → 2026-07-01]  Δ≥ [50 MB]  [清除] │
+---------------------------+----------------------+
|                           |  元数据面板 (30%)      |
|  文件树浏览器 (70%)       |  - 路径               |
|                           |  - 当前大小/增量       |
|  ~/                       |  - 文件数              |
|  ├── Desktop/  +1.2GB    |  - 修改时间            |
|  ├── Downloads/ -400MB   |  - 后缀分布(bar chart) |
|  │   ├── big_file.iso     |                       |
|  │   └── ...              +-----------------------+
|  └── Library/   +850MB    |  AI 智能观察窗 (30%)   |
|      └── Caches/          |  📦 Docker Buildx...  |
|                           |  🔍 用途说明...        |
|                           |  ⚠️ 删除影响...        |
|                           |  💡 建议: 一键清理     |
+---------------------------+----------------------+
```

**新增筛选栏**（第 2 行）：时间范围选择 + delta 阈值筛选 + 清除按钮。详见 §2.4。

### 2.2 列定义

默认三列：**名称** | **当前体积** | **体积增量 (Δ)**

- 增量列中，暴涨显示红色 `+1.2 GB`，减少显示绿色 `-400 MB`。
- 支持按任一列排序（按体积 / 按增量）。
- **筛选栏为空时**（无时间选择、无阈值）：增量列隐藏，文件树退化为两列：名称 + 体积。等同于 ncdu 模式。

### 2.3 Vim-like 快捷键映射

| 快捷键 | 功能 |
|--------|------|
| `j` / `k` | 上下移动光标 |
| `h` / `l` (或 `Enter`/`Backspace`) | 进入子目录 / 返回上级 |
| `s` | 扫描当前树根目录（首次使用或重新扫描） |
| `o` | 切换排序规则 |
| `d` | 触发安全删除流程 |
| `a` | 手动触发 AI 深度诊断 |
| `Tab` | 在筛选栏 / 文件树 / 元数据 / AI 面板间切换焦点 |
| `/` | 搜索文件/目录 |
| `?` | 显示帮助面板 |
| `q` / `Esc` | 退出 / 取消当前操作 |

### 2.4 筛选栏

筛选栏位于树上方，包含三个控件：

| 控件 | 作用 | 为空时 |
|------|------|--------|
| 时间范围 (from/to) | 选择要对比的两个时间点。独立模式下列出该路径的可用的快照时间戳；Daemon 模式下任意日期 | Delta 列隐藏，纯 ncdu |
| Delta 阈值 | 仅显示 `|delta| >= threshold` 的节点 | 不额外过滤（显示所有 delta，正的或负的） |
| 清除按钮 | 清空所有筛选条件，回到 ncdu 模式 | — |

筛选栏通过 `Tab` 聚焦后用左右方向键或 `Enter` 操作。选中时间框后显示日历或时间列表供选择。

### 2.5 智能交互

- **光标停顿 1 秒**：自动异步加载选中目录的 AI 诊断（如已配置 AI）。
- **实时进度条**：扫描或 Diff 计算时显示进度百分比。
- **编辑模式**：可复制 AI 推荐的修复命令直接执行。

## 3. GUI 端（跨平台现代桌面版 - 后期）

### 3.1 技术选型偏好

- **方案一 Slint**：纯 Rust 原生开发，超轻量，内存占用极小。
- **方案二 Tauri v2 + Frontend (Vue3/Vite)**：开发迅速，可引入 D3.js 绘制太阳图 (Sunburst Chart) 或矩形树图 (Treemap)。

### 3.2 核心特性

- 一键集成到 macOS 菜单栏 (MenuBar)。
- 日常作为小图标静默常驻。
- 触发空间变动阈值时弹窗气泡提醒。
- 可视化磁盘历史趋势折线图。
