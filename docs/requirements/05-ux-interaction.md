# 多端用户体验与交互设计

## 1. CLI 端（快速验证与自动化测试）

面向脚本、自动化管道和核心逻辑验证。

### 1.1 命令集

```bash
# 扫描目录并打印摘要（内存操作，不持久化）
argus scan --path ~/Downloads
```

### 1.2 阈值参数

`--threshold` 参数接受人类可读的体积格式，由 `argus-core` 提供解析函数 `parse_human_size`：

| 输入 | 实际值（字节） |
|------|---------------|
| `500B` | 500 |
| `10KB` | 10,240 |
| `50MB` | 52,428,800 |
| `2.5GB` | 2,684,354,560 |

默认值 `0` 表示显示所有变动（不过滤）。

> **Phase 1 实现**：`argus-core` 中实现 `parse_human_size` 函数，`argus-cli` 在解析参数时调用。

### 1.3 退出码契约

CLI 退出码标准化，支持脚本化管道和 CI 集成：

| 退出码 | 含义 | 适用场景 |
|--------|------|---------|
| `0` | 成功，未发现超阈值变动 | `argus scan` / `argus diff` 正常完成 |
| `2` | 参数错误 | 非法路径、缺失必选参数、格式错误 |
| `3` | IO 错误 | 快照文件不存在、权限不足、写入失败 |
| `4` | 内部错误 | 快照损坏、数据不一致、反序列化失败 |

## 2. TUI 端（Vim-like 核心极客版）

基于 `ratatui` 开发，Phase 2 实际布局分为三块，AI 面板留待 Phase 4。

### 2.1 布局与视觉

```
+--------------------------------------------------+
|  Argus v0.1.0              [?] Help  [Q] Quit    |
+---------------------------+----------------------+
|                           |  元数据面板 (30%)      |
|  文件树浏览器 (70%)       |  - 路径               |
|                           |  - 当前大小            |
|  ~/                       |  - modified time       |
|  ├── Desktop/  2.1 GB    |  - 修改时间            |
|  ├── Downloads/ 4.4 GB   |                       |
|  │   ├── big_file.iso     |                       |
|  │   └── ...              |                       |
|  └── Library/  1.2 GB     |                       |
|      └── Caches/          |                       |
+---------------------------+----------------------+
```

**未扫描目录**：无扫描汇总数据的目录在 size 列显示 `"-"`（灰色），文件始终显示真实大小。
**结构占位节点**：跳过目录中的深层子孙节点（如 `node_modules/pkg/src/`）显示 `"..."`（灰色），表示该节点是结构占位符，深层元数据未持久化；这类目录仍然可以继续展开。
**已扫描目录**：有扫描汇总数据后（按 `s` 扫描树根后），目录显示真实汇总大小。

### 2.2 列定义

默认两列：**名称** | **当前体积**
- 支持按名称或体积排序。
- **未扫描目录**：size 列显示 `"-"`（灰色），文件始终显示真实大小。
- **结构占位节点**：深层结构占位符显示 `"..."`（灰色），但目录仍可展开。

> 未来 daemon 模式下将新增第三列（体积增量 Δ）。

### 2.3 Vim-like 快捷键映射

| 快捷键 | 功能 |
|--------|------|
| `j` / `k` | 上下移动光标 |
| `h` / `l` (或 `Enter`/`Backspace`) | 进入子目录 / 返回上级 |
| `s` | 扫描当前树根目录（首次使用或重新扫描）；**不再弹出路径输入框** |
| `o` | 切换排序规则（名称/体积） |
| `d` | 触发安全删除流程 |
| `/` | 搜索文件/目录 |
| `?` | 显示帮助面板 |
| `q` / `Esc` | 退出 / 取消当前操作 |

### 2.4 时间筛选（Daemon 模式）

Phase 2（独立模式）**无时间筛选栏**。时间筛选（from/to 任意时间范围 + delta 阈值）将在 Phase 3（Daemon 模式）中实现，数据来源为 daemon 收集的增量事件，而非全量快照对比。

### 2.5 智能交互

- **启动行为**：启动时展示纯 FS 树（目录显示 `-`）。按 `s` 触发扫描后展示 size。
- **配置项**：`[browsing].auto_scan_on_start` 控制是否启动时自动扫描 cwd。
- **光标停顿 1 秒**：Phase 4 再启用的自动异步 AI 诊断触发。
- **实时进度条**：扫描或 Diff 计算时显示进度百分比。
- **编辑模式**：Phase 4 再启用的 AI 推荐修复命令执行。

## 3. GUI 端（跨平台现代桌面版 - 后期）

### 3.1 技术选型偏好

- **方案一 Slint**：纯 Rust 原生开发，超轻量，内存占用极小。
- **方案二 Tauri v2 + Frontend (Vue3/Vite)**：开发迅速，可引入 D3.js 绘制太阳图 (Sunburst Chart) 或矩形树图 (Treemap)。

### 3.2 核心特性

- 一键集成到 macOS 菜单栏 (MenuBar)。
- 日常作为小图标静默常驻。
- 触发空间变动阈值时弹窗气泡提醒。
- 可视化磁盘历史趋势折线图。
