# 项目概述与定位

## 1. 项目背景

在个人桌面端（macOS/Linux），用户经常遭遇"磁盘空间不足"的警告。系统自带的清理工具通常只显示模糊的"系统数据"，而传统的 `ncdu` 或 `du` 工具仅能提供当前的静态体积大小，无法感知"最近几天到底是谁在疯狂写入"。同时，面对 `/Library` 或 `/var` 下的怪异目录，普通用户由于"未知恐惧"通常不敢轻易删除。

## 2. 项目定位

### 2.1 核心定位

**面向个人桌面用户的智能磁盘健康治理工具**。

名称释义：
- **Argus**（TUI 客户端）：取自古希腊神话中的"百眼巨人"，象征巨细靡遗地监控磁盘。
- **Argusd**（守护进程）：后台常驻的守护进程。

### 2.2 明确不做

- **非企业服务器运维工具**：生产环境有 Prometheus/Grafana 监控、ELK 日志收集，且安全合规禁止第三方 AI 工具接入。
- **非传统重复造轮子**：不与 `ncdu` 全面竞争，核心差异化卖点是**后台守护进程增量监控**和**AI 治理**。

## 3. 目标用户群

| 用户类型 | 典型场景 | 核心诉求 |
|---------|---------|---------|
| 个人开发者 | Mac/PC 磁盘满，不知道谁在吃空间 | 找出最近暴涨的目录，安全清理 |
| 独立创作者 | 视频/设计缓存堆积 | 理解缓存目录用途，决定是否删除 |
| 普通桌面用户 | 系统提示磁盘空间不足 | 安全清理，不误删系统文件 |

## 4. 核心价值

1. **后台增量监控（daemon）**：揪出"过去一段时间内哪个目录在疯狂写入"，而非仅看当前静态大小。
2. **AI 百科**：为用户解释怪异目录的用途、能否删除、删除影响，消除"未知恐惧"。
3. **安全优先**：系统级保护 + 两阶段确认 + 废纸篓机制，杜绝误删。
4. **多端覆盖**：CLI（自动化/测试）→ TUI（极客）→ GUI（大众），渐进式演进。

## 5. 设计原则

- **AI 非必选**：用户无需配置 AI 即可完美使用传统扫描与浏览功能（参见 [06-ai-design.md](06-ai-design.md)）。
- **极致轻量**：守护进程闲置时 CPU ≈ 0%，内存 < 30MB。
- **隐私优先**：AI 仅上传目录名、路径和文件类型特征，绝不扫描或上传文件具体内容。
- **跨平台**：首阶段支持 macOS（Apple Silicon/Intel）和 Linux。
