# 安全设计

## 1. 设计理念

安全是 Argus 的红线。作为一个具备 AI 建议和删除能力的磁盘工具，必须在设计上杜绝误删系统关键文件的风险。

## 2. 系统防护层 (System Shield)

### 2.1 硬编码黑名单

以下目录在任何客户端（CLI/TUI/GUI）中：
- **不提供"删除"操作按钮/命令**。
- 即使通过 AI 建议窗口也不会推荐删除。
- 由 Rust 核心库在编译期/启动时硬编码。

**macOS 保护列表**：
```
/System
/System/Volumes
/usr/bin
/usr/lib
/bin
/sbin
/etc
/var/db
```

**Linux 保护列表**：
```
/boot
/etc
/dev
/proc
/sys
/usr/bin
/usr/lib
/bin
/sbin
/lib
/lib64
```

### 2.2 动态风险判定

除硬编码黑名单外，结合以下规则动态判定风险等级：

| 风险等级 | 判定条件 | 交互要求 |
|---------|---------|---------|
| 安全 | 用户个人目录下的非系统缓存 | 一次 Enter 确认 |
| 低风险 | 应用程序缓存目录 | 二次确认弹窗 |
| 中风险 | 系统级辅助目录（如 `/var/tmp`） | 输入 `yes` 确认 |
| 高风险 | 系统目录 / AI 判定为高风险 | 长按 3 秒 + 输入 `YES` |

## 3. 删除操作流程

### 3.1 标准删除流程

```
用户按 d
  → TUI 弹出确认框，展示：路径、大小、AI 风险等级
  → 用户确认
    → 低风险：直接移至废纸篓
    → 中风险：输入 yes 后再执行
    → 高风险：长按 3 秒 + 输入 YES
  → 操作成功/失败通知
```

### 3.2 废纸篓优先

- **macOS**：调用 `NSFileManager` 的 `trashItemAtURL` API 移至废纸篓。
- **Linux**：使用 `trash` crate 的 `trash::delete` API 调用桌面环境废纸篓（遵循 FreeDesktop Trash 规范）。
- 仅在废纸篓不可用或用户强制要求时使用硬删除。
- 硬删除使用安全删除模式（覆写后再删）作为可选配置。

### 3.3 删除回滚

- 废纸篓机制天然提供回滚能力。
- 硬删除操作记录日志（路径、时间、操作方式），方便审计。
- AI 建议的删除操作在执行前必须先经过安全护栏检查。

## 4. AI 安全护栏 (Safety Guard)

### 4.1 删除前安全检查

当用户尝试执行 AI 建议的删除操作前，系统强制执行：
1. 检查路径是否在黑名单中 → 禁止操作。
2. 检查路径是否在高风险模式 → 要求强交互。
3. AI 出具风险等级评定 → 展示给用户。

### 4.2 AI 争议与风险提示

当用户尝试删除某些高风险目录时，AI 作为"安全副驾驶"主动提示：

```
⚠️ AI 风险提示：
您正在尝试清理 target/ 目录。虽然能释放 15GB 空间，
但这是 Rust 编译缓存。由于您配置了 sccache，
清理后首次编译需重建本地索引，预计下次 cargo build
时间将从 5 秒延长至 3 分钟。是否确认删除？
```

## 5. 日志与审计

- 所有删除操作（包括废纸篓移动）均记录日志。
- 日志包含：操作时间、用户、路径、大小、操作方式、是否成功。
- 日志存储在 `~/.config/argus/audit.log`。
- 用户可通过 `argus audit` 命令查看操作历史。
